TECH STACK:
- Frontend: React.js (with functional components and hooks)
- Backend: Node.js with Express.js
- Database: MongoDB (Mongoose ODM)
- Authentication: JWT-based authentication
- Styling: Modern UI using CSS or Tailwind (clean cards, soft shadows, rounded corners)

- Implement login and registration APIs

- JWT token generation and protected routes
- Role-based redirection after login

2. ROLE-BASED ACCESS:
- Admin:
  - Create Managers and Workers
  - View all users and tasks
- Manager:
  - Create and assign tasks to Workers
- Worker:
  - View assigned tasks
  - Update task progress and status

3. DASHBOARD:
- Show summary cards:
  - Total Tasks
  - Completed Tasks
  - Pending Tasks
- Add a pie chart (Completed vs Pending)
- Responsive layout

4. TASK MANAGEMENT:
- Create Task with fields:
  - name
  - description
  - assignedTo (Worker ID)
  - progress (0–100)
  - status (Pending, In Progress, Completed)
  - dueDate
- Task list with progress bars
- Update task progress
- Ensure no undefined/null errors (proper default values & validation)

5. REPORTS PAGE:
- Show:
  - Total tasks
  - Completed tasks
  - Pending tasks
- Basic performance insights

6. UI LAYOUT:
- Sidebar navigation:
  - Dashboard
  - Users (Admin only)
  - Tasks
  - Reports
  - Logout
- Clean SaaS-style UI
- Fully responsive

7. BACKEND STRUCTURE (MVC):
- Models: User, Task
- Controllers: authController, taskController, userController
- Routes:
  - /api/auth
  - /api/tasks
  - /api/users
- Middleware:
  - authMiddleware (JWT verification)
  - roleMiddleware

8. DATABASE SCHEMA:

User:
- username (String)
- password (String)
- role (admin | manager | worker)

Task:
- name (String)
- description (String)
- assignedTo (ObjectId → User)
- progress (Number, default: 0)
- status (String)
- dueDate (Date)
- createdBy (ObjectId → User)
